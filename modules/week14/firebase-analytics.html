# Analytics 🚀

Firebase Analytics is a free app measurement solution that provides insights into app usage and user engagement. Key concepts include:

  • Events: Actions users take in the app (e.g., button clicks, page views).
  • User Properties: Attributes assigned to users (e.g., age, interests).
  • Standard vs. Custom Events: Firebase supports predefined events, but custom events allow flexibility.

# Tracking Events

Tracking events is crucial for understanding user behavior.

# Example 1: Tracking a Button Click Event

You can log custom events using Analytics.logEvent(_:parameters:).

import FirebaseAnalytics

func logButtonClickEvent(buttonName: String) async {
    Analytics.logEvent("button_click", parameters: [
        "button_name": buttonName,
        "timestamp": Date().timeIntervalSince1970
    ])
    print("Event logged: Button \(buttonName) clicked.")
}

# Setting User Properties

User properties describe characteristics of your app's users.

func setUserProperty(key: String, value: String) async {
    Analytics.setUserProperty(value, forName: key)

# Combining Firebase Authentication and Analytics

You can link Firebase Authentication data with Analytics events.

# Example: Logging Events with User ID

When a user logs in, associate their ID with analytics:

import FirebaseAuth

func setUserIdForAnalytics() async throws {
    guard let user = Auth.auth().currentUser else {
        print("No user is signed in.")
        return
    }

    Analytics.setUserID(user.uid)
    print("Analytics User ID set: \(user.uid)")
}

# Testing and Debugging

# Debug View in Firebase Console

You can use the Firebase DebugView to monitor events in real-time. For this you need to launch your app with this argument:

-FIRDebugEnabled
1

# Steps

  • Open your project in Xcode.
  • Select your project in the Project Navigator.
  • Go to the "Scheme" menu at the top of Xcode (next to the "Stop" button).
  • Select "Edit Scheme...".
  • In the dialog that appears:
  • Select the "Run" option in the left sidebar.
  • Click the "Arguments" tab.
  • Under "Arguments Passed On Launch", add -FIRDebugEnabled.
  • Press OK.

Here's a small SwiftUI app that demonstrates Firebase Analytics concepts: logging events, tracking page views, and setting user properties. It assumes Firebase is already integrated into the project.

# Small App Overview

The app contains:

  1. A Home Screen that logs page views.
  2. A Button to log custom click events.
  3. A User Properties Example to demonstrate setting properties like user preferences.

# Code Example

import SwiftUI
import FirebaseAnalytics

struct ContentView: View {
    var body: some View {
        NavigationView {
            VStack(spacing: 20) {
                Text("Firebase Analytics Demo")
                    .font(.headline)
                    .padding()

                // Button to log click events
                Button(action: {
                    Task {
                        await logButtonClickEvent(buttonName: "SubmitButton")
                    }
                }) {
                    Text("Log Button Click Event")
                        .padding()
                        .background(Color.blue)
                        .foregroundColor(.white)
                        .cornerRadius(10)
                }

                // Button to set user properties
                Button(action: {
                    Task {
                        await setUserProperty(key: "app_theme", value: "dark_mode")
                    }
                }) {
                    Text("Set User Property")
                        .padding()
                        .background(Color.green)
                        .foregroundColor(.white)
                        .cornerRadius(10)
                }

                // Navigation to another screen to log page views
                NavigationLink("Go to Details Screen") {
                    DetailsView()
                }
                .padding()
                .background(Color.orange)
                .foregroundColor(.white)
                .cornerRadius(10)
            }
            .padding()
            .onAppear {
                Task {
                    await logPageView(screenName: "HomeScreen")
                }
            }
        }
    }
}

struct DetailsView: View {
    var body: some View {
        Text("Details Screen")
            .font(.headline)
            .padding()
            .onAppear {
                Task {
                    await logPageView(screenName: "DetailsScreen")
                }
            }
    }
}

# Features Explained

  1. Page View Tracking

    • Logged automatically when screens appear (onAppear).

  2. Custom Events

    • Button clicks are logged with Analytics.logEvent.

  3. User Properties

    • Analytics.setUserProperty assigns attributes (e.g., preferred app theme).

# How to Test

  1. Run the App in Debug Mode

    • Enable debug mode with -FIRDebugEnabled (refer to steps in previous explanation).

  2. Trigger Events

    • Open the app and navigate between screens.
    • Click buttons to log events and set user properties.

  3. Verify Events in Firebase DebugView

    • Open the Firebase Console.
    • Navigate to Analytics → DebugView.
    • Monitor logged events in real time.
